# 接口文档：实战课程个人完课情况同步

## 1. 接口概述

**接口路径**：`POST /external-api/handsOnCoursesSync`

**接口描述**：对外提供实战课程个人完课情况的信息同步能力。调用方传入工号、课程类型、课程状态及备注，系统根据「工号 + 课程类型」判断数据库中是否已有记录：存在则更新，不存在则插入；更新或插入时自动写入当前时间到 `update_time` 字段。

**所属模块**：外部接口（External API）

## 2. 接口定义

### 2.1 基本信息

| 项目 | 说明 |
|------|------|
| URL | `/external-api/handsOnCoursesSync` |
| Method | POST |
| Content-Type | application/json |

### 2.2 请求参数（RequestBody）

请求体为 JSON，字段如下：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| account | String | 是 | 工号 |
| task_type | String | 是 | 课程类型 |
| task_status | String | 是 | 课程状态 |
| task_info | String | 否 | 课程备注 |

**说明**：
- `account`、`task_type`、`task_status` 为必填；`task_info` 为选填。
- 必填字段用于唯一定位与业务状态；选填字段 `task_info` 在更新时：若传入则覆盖原值，**未传或为空时则将库表该字段更新为空**（置空，不保留原值）。

### 2.3 请求示例

```json
{
  "account": "12345678",
  "task_type": "AI实战",
  "task_status": "已完成",
  "task_info": "2025年Q1完成"
}
```

## 3. 数据库表说明

### 3.1 表名

`hands_on_courses`（表名不带 `t_` 前缀）

### 3.2 涉及字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| account | String/Varchar | 工号 |
| task_type | String/Varchar | 课程类型 |
| task_status | String/Varchar | 课程状态 |
| task_info | String/Varchar/Text | 课程备注 |
| update_time | DateTime | 更新时间（由接口逻辑设置为当前时间） |

**说明**：若表中有主键或唯一键（例如 `(account, task_type)`），同步逻辑应基于该唯一维度进行「存在则更新，否则插入」。

## 4. 业务逻辑

### 4.1 处理流程

1. **参数校验**：校验必填参数 `account`、`task_type`、`task_status` 非空。
2. **查询是否存在**：根据 `account` + `task_type` 查询表 `hands_on_courses` 是否已有记录。
3. **分支处理**：
   - **存在**：更新该记录的 `task_status`、`task_info`，并将 `update_time` 置为当前时间 `now()`。其中 `task_info` 为选填：传入则覆盖，**未传或为空时更新为空**（不保留原值）。
   - **不存在**：插入新记录，字段为：`account`、`task_type`、`task_status`、`task_info`（可选，可为空）、`update_time`（`update_time` 使用 `now()`）。
4. **返回**：统一返回成功或失败及简要说明。

### 4.2 唯一性约定

- 判定「是否存在」的条件：`account` + `task_type`。
- 建议表中对该组合建立唯一约束或唯一索引，避免重复插入。

## 5. 响应结果

### 5.1 成功响应

**HTTP 状态码**：200

**响应体示例**：

```json
{
  "code": 200,
  "message": "同步成功",
  "data": {
    "action": "insert",
    "account": "12345678",
    "task_type": "AI实战"
  }
}
```

或更新时：

```json
{
  "code": 200,
  "message": "同步成功",
  "data": {
    "action": "update",
    "account": "12345678",
    "task_type": "AI实战"
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 200 表示成功 |
| message | String | 提示信息 |
| data | Object | 可选，便于调用方区分本次是新增还是更新 |
| data.action | String | 本次操作：`insert` / `update` |
| data.account | String | 工号 |
| data.task_type | String | 课程类型 |

（若不需要区分 insert/update，可简化为 `"message": "同步成功"`，`data` 为 null 或简单说明。）

### 5.2 失败响应

**参数校验失败（如缺少必填）**：

```json
{
  "code": 400,
  "message": "工号不能为空",
  "data": null
}
```

**服务器异常**：

```json
{
  "code": 500,
  "message": "系统异常：具体错误信息",
  "data": null
}
```

## 6. 注意事项与待确认

1. **表名与库表结构**：请确认最终表名（如是否带前缀 `t_`）以及各字段类型、长度、是否可空，与本文档一致后再开发。
2. **唯一键**：建议表上存在 `(account, task_type)` 唯一约束，便于「存在则更新否则插入」的实现与数据一致性。
3. **事务**：单条同步建议在 Service 层使用事务，保证更新/插入的原子性。
4. **幂等**：同一 `account` + `task_type` 多次调用应等价于一次调用（最后一次状态与备注为准），接口设计满足幂等。
5. **选填字段**：`task_info` 为选填。更新时若选填字段为空（未传或传空字符串），**需将库中该字段更新为空**，不保留原值。

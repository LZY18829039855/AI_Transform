# 前后端项目初始化框架规范（SPEC）

| 属性 | 说明 |
|------|------|
| 文档类型 | 技术规范（Specification） |
| 适用范围 | 基于 Spring Boot + Vue 3 的全栈 Web 应用 |
| 文档用途 | 定义可复用的项目脚手架、技术选型、工程约定与初始化流程 |
| 版本 | 1.0.0 |
| 最后更新 | 2026-07-09 |

---

## 1. 规范概述

### 1.1 目标

本规范定义一套**与业务无关**的前后端项目初始化标准，用于：

- 统一技术栈版本与依赖选型
- 统一工程目录结构与分层约定
- 统一数据库连接、部署路径、API 契约
- 提供可复制的新项目初始化流程

### 1.2 架构模式

采用**前后端分离 + 双仓库**模式：

```
{workspace}/
├── {backend-repo}/          # 后端：Spring Boot 服务
├── {frontend-repo}/         # 前端：Vue 3 SPA
│   ├── {frontend-app}/      # 前端主应用
│   └── mock-server/         # 可选：本地 Mock 服务
└── {workspace}.code-workspace  # 可选：IDE 多根工作区
```

### 1.3 占位符约定

本文档使用以下占位符，初始化新项目时须全局替换：

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{PROJECT_NAME}` | 项目英文名（kebab-case） | `my-app` |
| `{PROJECT_NAME_SNAKE}` | 项目英文名（snake_case） | `my_app` |
| `{FRONTEND_APP}` | 前端应用目录名 | `web-app` |
| `{GROUP_ID}` | Maven groupId | `com.example` |
| `{ARTIFACT_ID}` | Maven artifactId | `my-app-backend` |
| `{JAVA_PACKAGE}` | Java 根包名 | `com.example.myapp` |
| `{DB_NAME}` | 数据库名 | `my_app` |
| `{API_PREFIX}` | 后端 Context Path | `/my_app_webapi` |
| `{WEB_BASE}` | 前端部署子路径 | `/my_app/` |
| `{SERVER_PORT}` | 后端端口 | `8080` |
| `{JAVA_VERSION}` | JDK 版本 | `1.8` |

| `test` | `DOCKER_ENV=test` | `application-test.yml` |
| `prod` | `DOCKER_ENV=prod` | `application-prod.yml` |

### 3.7 工程约定

| 约定项 | 规范值 |
|--------|--------|

### 4.5 工程约定

| 约定项 | 规范值 |
|--------|--------|
| 部署 Base Path | `{WEB_BASE}`（建议格式：`/{project}/`，首尾带 `/`） |

| SCSS 全局变量 | 通过 `vite.config.ts` 自动注入 |

### 4.6 HTTP 请求封装规范

```typescript
class Request {
  constructor(private baseURL: string, private timeout = 10000) {}

  async request<T>(url: string, options?: RequestInit): Promise<T> {
    const headers = new Headers({ 'Content-Type': 'application/json' })
    const token = localStorage.getItem('token')
    if (token) headers.set('Authorization', `Bearer ${token}`)
    // fetch + timeout + error handling
  }
}

export const request = new Request('{API_PREFIX}')
export const get = <T>(url: string) => request.request<T>(url, { method: 'GET' })
export const post = <T>(url: string, body?: unknown) =>
  request.request<T>(url, { method: 'POST', body: JSON.stringify(body) })
```

### 4.7 路由规范

- 使用 Vue Router 4 History 模式
- 路由按功能模块拆分，`meta` 字段承载 `title`、`requiresAuth` 等元信息
- 全局 `beforeEach` 守卫处理鉴权与权限校验
- 页面组件使用动态 `import()` 懒加载

### 4.8 Nginx 配置模板

```nginx
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;

    location = {WEB_BASE_TRIM} {
        return 301 {WEB_BASE};
    }

    location {WEB_BASE} {
        try_files $uri $uri/ {WEB_BASE}index.html;
    }

    location ~* ^{WEB_BASE}.*\.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

> `{WEB_BASE_TRIM}` = 去掉尾部 `/` 的 `{WEB_BASE}`，例如 `/my_app`


### 5.7 表命名约定

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 业务主表 | `t_{entity}` | `t_user` |
| 同步表 | `t_{entity}_sync` | `t_user_sync` |

### 5.8 MyBatis 映射约定

- Mapper 接口与 XML 文件同名（如 `UserMapper.java` ↔ `UserMapper.xml`）

- 复杂查询使用 XML，简单 CRUD 可使用注解（按需）
- 字段映射遵循下划线 → 驼峰自动转换

## 6. 前后端联调规范

### 6.1 请求链路

```
浏览器
  │
  ├─ 开发环境 ──► Vite Dev Server (:5173)
  │                  │
  │                  └─ proxy {API_PREFIX} ──► Spring Boot (:{SERVER_PORT})
  │
  └─ 生产环境 ──► Nginx (:80) {WEB_BASE}
                     │
                     └─ 反向代理 {API_PREFIX} ──► Spring Boot (:{SERVER_PORT})
```

### 6.2 契约对照

| 约定 | 后端 | 前端 |
|------|------|------|
| API 前缀 | `server.servlet.context-path={API_PREFIX}` | `request.baseURL='{API_PREFIX}'` |
| 页面前缀 | — | `vite base='{WEB_BASE}'` |
| 响应结构 | `Result<T>` JSON | `fetch` → `response.json()` |
| 成功码 | `code: 200` | 按 `code` 判断 |
| 跨域 | `CorsConfig` 全局配置 | 开发走 Proxy，生产同域部署 |

### 6.3 完整 URL 示例

```
# 后端 API
http://localhost:{SERVER_PORT}{API_PREFIX}/api/users

# 前端页面（开发）
http://localhost:5173{WEB_BASE}

# 前端页面（生产）
http://{domain}{WEB_BASE}
```

## 7. 新项目初始化流程

### 7.1 初始化检查清单

- [ ] 确定并替换全部占位符（见 1.3 节）
- [ ] 创建后端 Maven 工程并引入依赖
- [ ] 创建前端 Vite + Vue 3 + TS 工程并安装依赖
- [ ] 配置 `application.yml` 数据库连接
- [ ] 执行 `schema.sql` 建库
- [ ] 配置 `vite.config.ts` 的 `base` 与 `proxy`
- [ ] 配置 `nginx.conf` 子路径 SPA 路由
- [ ] 验证前后端联调
- [ ] 配置 Docker 构建（按需）
- [ ] 配置 CI 流水线（按需）

### 7.2 后端初始化步骤

```bash
# 1. 使用 Spring Initializr 或复制脚手架模板创建工程

# 2. 配置 pom.xml（见 3.1、3.2 节）

# 3. 创建包结构（见 3.3 节）
#    替换 {JAVA_PACKAGE} 为实际包名

# 4. 编写 application.yml（见 3.5 节）
#    替换 {DB_NAME}、{API_PREFIX}、{SERVER_PORT}

# 5. 初始化数据库
mysql -u root -p < src/main/resources/db/schema.sql

# 6. 启动验证
mvn spring-boot:run
curl http://localhost:{SERVER_PORT}{API_PREFIX}/health
```

## 9. 框架决策记录

以下技术决策作为标准基线，新项目默认沿用，变更需有明确理由：

| # | 决策 | 理由 |
|---|------|------|
| 1 | Spring Boot 2.7 + Java 8 | 企业内网部署兼容性 |
| 2 | MyBatis XML Mapper | 复杂 SQL 可读性与可维护性 |
| 3 | HikariCP + ENC 凭据加密 | 连接池性能 + 配置安全基线 |
| 4 | 统一 Result 响应 + 全局异常处理 | 稳定的前后端契约 |
| 5 | Vue 3 + Vite 5 + Element Plus 2 | 成熟的管理后台 / 看板技术组合 |
| 6 | fetch 封装替代 axios | 减少依赖，满足标准 REST 场景 |
| 7 | 子路径部署（`{WEB_BASE}`） | 适配网关 / Ingress 二级路径 |
| 8 | Pinia 模块化 + composables | 状态与逻辑复用 |
| 9 | 多阶段 Docker 构建 | 前端 Node 构建 + Nginx 静态托管 |
| 10 | 双仓库前后端分离 | 独立迭代、独立部署 |
